# RevvFi Documentation

Welcome to the official technical documentation for **RevvFi** — a liquidity-backed token launch protocol for trustless, LP-governed launches.

## What is RevvFi?

RevvFi is a blockchain protocol that enables fair, transparent token launches governed by liquidity providers. Instead of centralized control, LPs pool ETH, receive governance shares, and collectively decide on fund allocation and project parameters.

### Key Features

- **LP-Governed Launches** - Liquidity providers control treasury and strategic funds
- **Per-Token Isolation** - Each launch has independent contracts and parameters  
- **Internal Share Ledger** - No separate governance token needed
- **Trustless Refunds** - Automatic ETH refunds if launch fails
- **Popularity Scoring** - On-chain metrics help LPs evaluate opportunities
- **Creator Vesting** - Multi-sig protection with cliff + linear schedules

## Quick Start

### For Developers

```bash
git clone https://github.com/RevvFi/revvfi-docs.git
cd revvfi-docs
pnpm install
pnpm dev
```

Visit `http://localhost:3000`

1. Review [Architecture](/architecture) to understand contract design
